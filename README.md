# torrentd

Мини-клиент BitTorrent на Go. Учебный pet-проект: демон, который парсит
`.torrent`-файл, обращается к трекеру, скачивает куски от пиров параллельно
(горутина на пира) и сверяет их SHA-1 хэши.

Развивается в **демон + gRPC + TUI**: фоновый движок держит несколько раздач и
их состояние, наружу выставляет gRPC, а управляет им консольный клиент на
bubbletea.

> Проект пишется вручную, с нуля, ради изучения Go: бинарные форматы, сетевые
> протоколы, конкурентность и синхронизация. Ядро (bencode → скачивание) —
> только стандартная библиотека; сетевой фасад добавляет grpc-go.

## Как это работает

```
.torrent (bencode) ──► метаданные ──► трекер (HTTP) ──► список пиров
                          │                                   │
                     info_hash                     горутина на пира ──► TCP
                          │                                   │
                          └── SHA-1 хэши кусков   хендшейк / bitfield / request / piece
                                                              │
                                              проверка SHA-1 куска ──► запись на диск
```

## Структура

```
cmd/torrentd/          точка входа (демон)
internal/
  bencode/             декодер формата bencode
  torrentfile/         разбор .torrent, вычисление info_hash
  tracker/             запрос к трекеру, получение списка пиров
  peer/                протокол общения с пиром (TCP, хендшейк, сообщения)
  download/            движок: Manager (состояние торрентов) + скачивание кусков
  config/              настройки
proto/                 gRPC-контракт (.proto)          [в работе]
internal/rpc/          сгенерённый код gRPC             [в работе]
internal/rpcserver/    реализация сервера поверх Manager [в работе]
```

Направление зависимостей — в одну сторону, без циклов:
`rpcserver → download → peer → tracker → torrentfile → bencode`.

`download.Manager` держит карту раздач и их прогресс под мьютексом; каждая
раздача качается в своей горутине, состояние безопасно читается снаружи.

## Прогресс

- [x] **bencode** — декодер (числа, строки, списки, словари), счётчик байтов
      для захвата сырых данных `info`
- [x] **torrentfile** — разбор `.torrent`, `info_hash`, нарезка хэшей кусков
- [x] **tracker** — сборка announce-URL, HTTP-запрос, разбор ответа
- [x] **peer** — TCP-хендшейк, обмен сообщениями, bitfield, скачивание кусков
- [x] **download** — параллельное скачивание кусков (горутины + каналы) и сверка SHA-1
- [x] **демон** — `Manager` с состоянием раздач, фоновая закачка, потокобезопасный прогресс
- [ ] **gRPC** — фасад над `Manager` (`AddTorrent` / `ListTorrents` / `WatchProgress`-стрим)
- [ ] **TUI** — консольный клиент на bubbletea поверх gRPC
- [ ] запись на диск через `WriteAt` (не держать файл в RAM), переспрос трекера, многофайловые раздачи

## Запуск

Из корня проекта (там, где `go.mod`):

```sh
go run ./cmd/torrentd <путь/к/файлу.torrent>
```

Пример:

```sh
go run ./cmd/torrentd ./internal/torrentfile/testdata/debian-13.6.0-amd64-netinst.iso.torrent
```

Детектор гонок (полезно для демона с горутинами):

```sh
go run -race ./cmd/torrentd <путь/к/файлу.torrent>
```
